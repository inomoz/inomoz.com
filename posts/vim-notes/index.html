VIM Notes

Here I collect, various VIM tips, shortcuts, keybindings, etc. Also, here some information how I handle daily usage of text editing.

This not absolute beginner cheatsheet, just some short basic commands/tips.

Recommendation - just use it & check it every day (training)

Base

Navigation

h j k l cursor move. This also work in Vimium, NNN, Zathura and other programs with Vim keyboard shortcuts.

w b move to Word <- ->.
w - mv/cp as (NNN)
b - bookmark (Vimium/NNN)

]w ]b move to Word <- -> also, but support CamelHumps and separated_word words

W B move to Word (ignore punctuation like .,) <- ->.
B - open bookmark in new tab (Vimium).

e ge move to worD <- ->
e - edit file (NNN)

^ g_ move to ..^lineg_.. <- ->. I think this is more useful than next shortcuts.
0 $ move to 0..line..$ <- ->

^e ^y move document ↑ ↓. Sometimes useful if need check few lines above/below.

{ } paragraph ↑ ↓. Also useful with V selection.

Go to

:[num] to line number

f F [char] to Char in line <- ->. Upper - back direction.
f - Open link in current tab (Vimium).
F - Open link in new tab and not change active tab (Vimium).

t T [char] to tchar in line <- ->. More useful than f/F if need for example delete from char to quote.

, ; [char] repeat search <- ->

% jump between () or {}. Append arguments? Or add code?

^o ^i go to older/newer jump item in jump list (also work in Zathura). You can check this list by :jumps command.
A jump is one of the following commands: '', ```, G, /, ?, n, N, %, (, ), [[, ]], {, }, :s, :tag, L, M, H and the commands that start editing a new file.

^d ^u up/down half page ↑ ↓. But this is not universal, I use custom keyboard and FNd FNu keys (works everywhere!).

gg G home / end ↑ ↓. gg is like go go go (starting). G is like Gone.

gT gt go to <- tab -> (also work in Vimium).

gv - reselect(select last selected block of text)

Editing text

i a enter edit i<-cursor->a. i - left side of cursor/character, a - right side of cursor/character.

I A enter edit I..<-line->..A

o O enter edit ↑↓..line..

X x cut char <- -> use this instead vd, d[motion] cut motion, dd cut line

c[motion] cut & edit motion, cc - line

r[char] replace char, R replace mode

J join line ↑, if you see 1 empty line below, just use 3J

y[motion] yank motion, yy line (also work in Vimium - copy current url)

p P paste <- -> ↑ ↓ depends on clipboard content (also work in Vimium - open clipboard URL).

^r+0 - in insert mode inserts the last yanked text, working in command mode

^r+" - in insert mode inserts clipboard, working in command mode

^o[command] - run command in insert mode once

Delete text (not cut)

'd[motion] delete text with motion. 'dd delete line, 'dw delete word.
This bindings require custom remap in vimrc file: noremap ' "_

'd[P] delete selected text (optional paste text)

Targets

All targets are currently just one character.

(, ), {, }, [, ], <, and >

The targets b, B, r, and a are aliases for ), }, ], and > ‘, “, `,

A t is a pair of HTML or XML tags.

The letters w, W, and s correspond to a |word|, a |WORD|, and a |sentence|.

A p represents a |paragraph|

Argtextobj

Additional text objects: aa - argument, ia - inner argument

Examples:

function(1, (20<press 'daa' here>*30)+40, somefunc2(3, 4))

function(1, (20<press 'cia' here>*30)+40, somefunc2(3, 4))

Exchange

To exchange two words, place your cursor on the first word and type cxiw. Then move to the second word and type cxiw again. Note: the {motion} used in the first and second use of cx don’t have to be the same.

cx[motion] Base exchange

cxx Like cx, but use the current line.

X Like cx, but for Visual mode.

cxc Clear any {motion} pending for exchange.

Some notes

If you’re using the same motion again (e.g. exchanging two words using cxiw), you can use . the second time.

If one region is fully contained within the other, it will replace the containing region.

Swap chars (built-in)

xp Swap the current character (the character under the cursor) with the next

Xp Swap the current character with the previous

Text objects for entire buffers

ae targets the entire content of the current buffer.

ie is similar to ae, but ie does not include leading and trailing empty lines
